# Prompt Pattern — Instagram Carousel Generation Instructions

## Before Generating

Read these config files and use their data throughout:
- `config/brand.yaml` — product info, voice, differentiators
- `config/platforms/instagram.yaml` — platform constraints and visual style
- `config/icps/{icp}.yaml` — target audience profile

## What You're Creating

A 5-6 slide Instagram carousel that teaches one reusable prompting technique. Educational, deep, save-worthy. Each pattern is a transferable skill the reader can apply immediately.

## Content Rules

1. **One pattern per carousel.** Never combine techniques.
2. **Name the pattern** — it should feel like a named concept (e.g., "Bounded Quantities", "The Role Frame", "End-Positioning").
3. **Show before and after** — demonstrate the pattern with a concrete example.
4. **Make it actionable** — the reader should be able to apply this technique in their next prompt.
5. **40-60 words max per slide** for mobile readability.

## Slide-by-Slide Structure

### Slide 1 — The Hook
**Purpose:** Name the pattern and create curiosity.

Content:
- Pattern name in large text
- One-line hook that makes you want to learn it
- Example hooks:
  - "One word changes everything."
  - "This trick reduces your token cost by 37%."
  - "You're putting the most important instruction in the wrong place."

### Slide 2 — The Problem
**Purpose:** Show what goes wrong without this pattern.

Content:
- The common mistake most people make
- What happens when you DON'T use this pattern
- Keep it relatable — "You've probably done this"

### Slides 3-4 — How It Works
**Purpose:** Teach the technique with a concrete example.

Content per slide:
- The technique explained in plain language
- A real before/after example showing the pattern applied
- If model-specific: note which models this works best for

### Slide 5 — Before vs After
**Purpose:** Side-by-side proof that the pattern works.

Content:
- Before: the common approach (labeled)
- After: the pattern applied (labeled)
- The difference should be immediately visible

### Slide 6 (optional) — When to Use It
**Purpose:** Help the reader know when to apply this pattern.

Content:
- 2-3 specific scenarios where this pattern is most effective
- Any models where it's especially important
- Subtle branding (handle only)

## Caption

1. **Hook** (first line): Provocative statement about the pattern.
2. **Teach** (3-4 sentences): Explain the pattern's core insight and why it works.
3. **Example** (1-2 sentences): A quick application the reader can try right now.
4. **CTA**: "Save this for your next prompting session." or "Which pattern should we cover next?"

## Hashtags

15 hashtags (5 high-volume, 5 mid-volume, 5 niche).

## Alt Text

Screen-reader descriptions for each slide image.

## Quality Rules

- Every pattern must be grounded in real model behavior (from PO's model guides)
- No hype language — the technique speaks for itself
- The before/after must be a real, meaningful improvement — not cosmetic
- Educational tone — this should feel like learning, not being sold to
