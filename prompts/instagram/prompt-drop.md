# Daily Prompt Drop — Instagram Single Image Generation Instructions

## Before Generating

Read these config files and use their data throughout:
- `config/brand.yaml` — product info, voice, differentiators
- `config/platforms/instagram.yaml` — platform constraints and visual style
- `config/icps/{icp}.yaml` — target audience profile

## What You're Creating

A single Instagram image (1080x1080) featuring one polished, ready-to-use prompt template optimized for a specific AI model. People screenshot and save these.

## Content Rules

1. **One prompt per post.** Complete, copy-paste ready.
2. **Specify the target model.** Each prompt is optimized for a specific model (ChatGPT, Claude, Gemini, Cursor, or Claude Code).
3. **Use model-specific formatting.** ChatGPT: Markdown. Claude: XML tags. Gemini: XML with end-weighted constraints. Cursor: @-references. Claude Code: verification-first.
4. **The prompt must be genuinely useful** — something the reader would actually use in their workflow.
5. **Keep it readable at phone size.** 60-80 words max on the image.

## Output Structure

### Image Content

**Heading** (large, white, uppercase):
"DAILY PROMPT DROP" or "PROMPT OF THE DAY"

**Model tag** (small label):
"Optimized for [MODEL]"

**The prompt** (black text on coral, or white text on black):
The actual prompt, formatted for the target model. Must be complete and copy-paste ready.

### Caption

1. **Hook**: What task this prompt solves and why it's better than what you'd normally type.
2. **Why this structure works** (2-3 sentences): Explain the model-specific formatting choices.
3. **How to customize**: Tell the reader what to swap out for their use case (brackets indicate replaceable sections).
4. **CTA**: "Save this for your next [task] session." or "Screenshot this — you'll need it."

### Hashtags

15 hashtags (5 high-volume, 5 mid-volume, 5 niche).

### Alt Text

Describe the image: background color, prompt text content, model tag, layout.

## Quality Rules

- The prompt must follow the model-specific rules from PO's model guides
- Use [brackets] for parts the user customizes
- No product pitching — the prompt itself is the value
- The prompt should feel like it was written by someone who deeply understands the model
