# User Story — Instagram Carousel Generation Instructions

## Before Generating

Read these config files and use their data throughout:
- `config/brand.yaml` — product info, voice, differentiators
- `config/platforms/instagram.yaml` — platform constraints and visual style
- `config/icps/{icp}.yaml` — target audience profile

## What You're Creating

A 5-slide Instagram carousel telling a real user's story of solving a problem with Prompt Optimizer. Social proof meets education. Shows the product in action through narrative, not advertisement.

## Content Rules

1. **Real stories only.** Use actual user experiences provided by the user. Never fabricate testimonials.
2. **Problem-first narrative.** Lead with the struggle, not the product.
3. **The product is the turning point, not the hero.** The user is the hero.
4. **Show the actual prompts.** Before and after. The proof is in the comparison.
5. **40-60 words max per slide** for mobile readability.

## Slide-by-Slide Structure

### Slide 1 — The Problem (Hook)
**Purpose:** Make the reader identify with the struggle.

Content:
- The user's pain point in relatable language
- Hook line that creates empathy
- Example hooks:
  - "20 minutes on one prompt. The output was still wrong."
  - "Same prompt. Claude nailed it. ChatGPT gave me garbage."
  - "I was about to give up on AI-assisted coding."

Design: Coral background, black/white text.

### Slide 2 — The Old Way
**Purpose:** Show what they were doing before — the frustration.

Content:
- How they were prompting before (the raw, unoptimized approach)
- The results they were getting (regenerate loop, mediocre output)
- Keep it specific — real task, real failure

Design: Cream background, black text, coral sidebar.

### Slide 3 — The Switch
**Purpose:** The moment they tried Prompt Optimizer.

Content:
- What prompted them to try it (optional — can skip if story flows better)
- The optimized prompt they got — show the actual structured prompt
- Which model it was optimized for

Design: Black background, coral heading, white text, coral sidebar.

### Slide 4 — The Result
**Purpose:** Show the payoff — what changed.

Content:
- The output quality difference
- Time saved, tokens saved, or iterations eliminated
- A direct quote from the user if available
- The optimized prompt displayed cleanly

Design: Coral background, white heading, black text.

### Slide 5 — The Takeaway
**Purpose:** The lesson and subtle CTA.

Content:
- One-line insight from the experience
- What the user does differently now
- Subtle branding (handle only)

Design: Cream background, coral heading, black text, coral sidebar.

## Caption

1. **Hook**: The user's problem restated as a universal pain point.
2. **The story** (3-4 sentences): Condensed narrative — problem → attempt → Prompt Optimizer → result.
3. **The insight** (1-2 sentences): What this story teaches about prompting.
4. **CTA**: "Have a prompt story? DM us." or "What's your worst prompting moment?"

## Hashtags

15 hashtags (5 high-volume, 5 mid-volume, 5 niche).

## Alt Text

Screen-reader descriptions for each slide image.

## Quality Rules

- NEVER fabricate user stories — all stories must be sourced from real users
- If no real story is available, the user must provide the story details
- The narrative should feel authentic, not polished marketing copy
- Product mention is limited to the turning-point slide — not every slide
- The user's voice should come through, not the brand's
- Educational value: every story should teach something about prompting
