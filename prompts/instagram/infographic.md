# Prompt Infographic — Instagram Single Image Generation Instructions

## Before Generating

Read these config files and use their data throughout:
- `config/brand.yaml` — product info, voice, differentiators
- `config/platforms/instagram.yaml` — platform constraints, NotebookLM config, visual style
- `config/icps/{icp}.yaml` — target audience profile

## What You're Creating

A single Instagram infographic image (portrait) packed with reference-worthy information: cheat sheets, comparison charts, tip compilations, or frameworks. Dense but readable. Designed to be saved and returned to.

The image is generated by NotebookLM. The caption is written by you.

## Interactive Workflow (Two Steps)

### Step 1: Topic Selection

When the user asks for an infographic post:

1. Pull 3 topic ideas — mix from the content bank in `content-strategy.md` and fresh ideas based on the NotebookLM research notebook
2. Present all 3 with a one-line pitch for each
3. Wait for the user to pick one

### Step 2: Generate Everything

Once the user picks a topic:

#### 2a. Research via NotebookLM

```bash
# Set notebook context (ID from config/platforms/instagram.yaml → notebooklm_integration.notebook_id)
notebooklm use <notebook_id>

# Query for deep, factual content on the topic
notebooklm ask "<specific research question tailored to the topic>" --json
```

Research query examples by infographic type:
- **Cheat sheet**: "What are the most important rules and best practices for [topic]? Give specific, actionable items with brief explanations."
- **Comparison**: "Compare [A] vs [B] in terms of [criteria]. What are the concrete structural and behavioral differences a practitioner would notice?"
- **Framework**: "What is the recommended mental model or framework for [topic]? Break it into clear stages, layers, or components with explanations."
- **Data insight**: "What specific data, benchmarks, or measured results exist about [topic]? Include numbers, percentages, and sources."

#### 2b. Write the Caption

Using the NotebookLM research + brand voice from `config/brand.yaml`, write:

1. **Hook** (1 line): A punchy, scroll-stopping opener. Not clickbait — real insight that earns the stop. Pattern: bold claim, surprising stat, or direct challenge.
2. **Body** (1 paragraph): Deep, research-backed context. This is NOT fluff. Pack it with specific facts, real comparisons, and technical insight pulled from the NotebookLM research. Write like someone who has actually tested this, not someone summarizing a blog post. Every sentence should teach something. Target 3-5 sentences.
3. **CTA** (1 line): "Save this for your next [specific use case]." or "Screenshot this before your next [task]." — specific, not generic.

**Quality bar**: The caption should make someone who knows prompt engineering nod and save it. If it reads like generic AI advice, rewrite it.

**Formatting rule**: Never use em dashes (—) in captions. Use periods, commas, or restructure the sentence instead.

#### 2c. Write Hashtags

15 hashtags total:
- 5 high-volume (>100K posts): broad AI/tech tags
- 5 mid-volume (10K-100K): prompt engineering specific
- 5 niche (<10K): Prompt Optimizer and tool-specific

#### 2d. Generate the Infographic Image

```bash
# Pick style based on infographic type (from config: notebooklm_integration.generation.style_by_type)
# cheat_sheet → professional, comparison → editorial, framework → instructional, data_insight → editorial
notebooklm generate infographic --orientation square --detail detailed --style <style> --json
```

Parse the `task_id` from the JSON output, then wait:

```bash
notebooklm artifact wait <artifact_id>
```

Download once complete:

```bash
notebooklm download infographic generated_slides/<slug>/infographic.png
```

#### 2e. Save Output

Save the complete post as markdown to:
```
output/instagram/infographic/{YYYY-MM-DD}_{slug}.md
```

## Output Markdown Format

```markdown
---
platform: instagram
category: infographic
topic: "<topic>"
icp: <icp>
date: <YYYY-MM-DD>
status: draft
infographic_image: generated_slides/<slug>/infographic.png
notebooklm_notebook: <notebook_id>
---

# <Infographic Title>

## Caption

<hook line>

<one deep paragraph — research-backed, specific, technical but accessible>

<CTA line>

## Hashtags
<15 space-separated hashtags>

## Alt Text
<Full description of the infographic for accessibility: title, content summary, layout, colors>

## Research Notes
<Key facts and citations from NotebookLM query — kept for reference/verification>
```

## Quality Rules

- Every fact in the caption must come from the NotebookLM research or be verifiably accurate
- The hook must earn the stop — no "Did you know..." or "Here's why..." openers unless they genuinely surprise
- The body paragraph is the value — it should feel like getting a free consulting session, not reading a social media post
- Voice: confident, technical but accessible, slightly irreverent. Never preachy or hypey.
- No product pitching in the caption — the information is the value. Handle (@PROMPTOPTIMIZER) is on the infographic image only.
- Match the ICP's language patterns and pain points from the ICP config
