# Prompt Optimizer — Marketing Content Pipeline

This project generates social media marketing content for Prompt Optimizer using Claude Code as the generation engine.

## How This Pipeline Works

All content is generated by Claude Code directly. Config files provide brand context, platform rules, and audience profiles. Prompt templates define the structure for each post type. No external APIs or scripts needed.

## Generation Workflow

When asked to generate a post, follow these steps **exactly**:

### 1. Parse the Request
Identify from the user's message:
- **Platform**: which social platform (e.g., `instagram`). Default: `instagram`
- **Category**: which post type (e.g., `autopsy`, `architecture`)
- **Topic**: the subject/prompt to build the post around
- **ICP**: which audience profile to target. Default: `solo-builder`

### 2. Load Context (read all 3 config files)
Before generating ANY content, read these files in order:
1. `config/brand.yaml` — product info, positioning, voice, differentiators
2. `config/platforms/{platform}.yaml` — platform constraints, category rules, formatting guidelines
3. `config/icps/{icp}.yaml` — target audience profile, pain points, language patterns

### 3. Load the Prompt Template
Read the generation instructions at:
```
prompts/{platform}/{category}.md
```
This file defines the exact output structure, slide-by-slide or section-by-section.

### 4. Generate Content
Using the loaded brand context, platform constraints, ICP profile, and prompt template instructions — generate the full post content. Follow the template structure exactly.

**Interactive categories (e.g., `autopsy`)**: Some categories have a two-step workflow:
- **Step 1**: Generate the initial content (bad prompt, dissection slides) and **pause**. Present the bad prompt to the user and ask them to provide the optimized version (from Prompt Optimizer).
- **Step 2**: Once the user provides the optimized prompt, insert it into the "Optimized Version" slide exactly as given, then complete the remaining sections (payoff slide, caption, hashtags, alt text).

Check the prompt template for whether it specifies an interactive workflow.

### 5. Write Output
Save the generated content as a markdown file to:
```
output/{platform}/{category}/{YYYY-MM-DD}_{slug}.md
```
Where:
- `{YYYY-MM-DD}` is today's date
- `{slug}` is the topic converted to lowercase kebab-case (e.g., "fix my code" → "fix-my-code")

If a file with the same name already exists, append a number: `fix-my-code-2.md`

### 6. Generate Slide Images (autopsy category)
After writing the markdown file, run the image generation script:
```
python3 scripts/generate_slides.py output/{platform}/{category}/{filename}.md
```
This creates a folder at `generated_slides/{filename}/` with one PNG per slide (1080x1350, dark background + white text). Each image is named `slide_01_hook.png`, `slide_02_model_hears.png`, etc.

## Output Markdown Format

Every generated post MUST use this frontmatter + section structure so it maps cleanly to Buffer for scheduling later:

```markdown
---
platform: instagram
category: autopsy
topic: "fix my code"
icp: solo-builder
date: 2026-03-11
status: draft
---

# [Post Title]

## Slides
(for carousels — one subsection per slide)

### Slide 1 — [Slide Title]
[slide content]

### Slide 2 — [Slide Title]
[slide content]

...

## Visual Description
(for infographics — design instructions for the designer)

## Text Content
(for infographics — all text that appears on the image)

## Caption
[Instagram caption text]

## Hashtags
[space-separated hashtags]

## Alt Text
[Accessibility descriptions]
```

## Rules

1. **Always read configs before generating.** Never generate from memory alone — the config files are the source of truth.
2. **Follow the prompt template structure exactly.** The templates define slide counts, section order, and content rules.
3. **Voice consistency.** Match the tone defined in `config/brand.yaml`. Confident, technical but accessible, slightly irreverent. Never preachy or hypey.
4. **ICP targeting.** Content must speak to the specific ICP's pain points and language patterns. Don't write generic content.
5. **Product mentions are minimal.** Most content is educational. Product branding is subtle — typically only on the final slide or as a small CTA.
6. **For autopsy posts: the optimized prompt comes from the user.** Never generate the optimized prompt yourself — always pause and ask the user to provide it from Prompt Optimizer. Use it exactly as given.

## Buffer Integration (Phase 2)

Currently, generated posts are saved as markdown files for manual review and upload to Buffer (publish.buffer.com/create/templates).

When Buffer API token is configured in `config/buffer.yaml`:
- **Template fetching**: Pull Buffer templates to discover new post formats for future categories
- **Scheduling**: Push generated content directly to Buffer as draft posts

## Adding New Content

| To add...               | Do this                                                                  |
|-------------------------|--------------------------------------------------------------------------|
| New platform (LinkedIn) | Create `config/platforms/linkedin.yaml` + `prompts/linkedin/{category}.md` |
| New ICP                 | Create `config/icps/{name}.yaml` following the solo-builder schema       |
| New post category       | Add category to platform yaml + create `prompts/{platform}/{category}.md`  |
