# Prompt Optimizer — Marketing Content Pipeline

This project generates social media marketing content for Prompt Optimizer using Claude Code as the generation engine.

## How This Pipeline Works

All content is generated by Claude Code directly. Config files provide brand context, platform rules, and audience profiles. Prompt templates define the structure for each post type. No external APIs or scripts needed.

## Content Categories

Six content categories, 3 posts per day:

| Category | Format | Frequency | Slot |
|----------|--------|-----------|------|
| **Prompt Autopsy** | Carousel (7 slides) | 4x/week | Afternoon |
| **Did You Know** | Single image | 5x/week | Morning |
| **Daily Prompt Drop** | Single image | 4x/week | Morning/Evening |
| **Prompt Pattern** | Carousel (5-6 slides) | 3x/week | Afternoon |
| **Prompt Infographic** | Single image (1080x1350) | 3x/week | Evening |
| **User Story** | Carousel (5 slides) | 2x/week | Evening |

See `content-strategy.md` for the full posting calendar, content banks, and weekly rotation.

## Generation Workflow

When asked to generate a post, follow these steps **exactly**:

### 1. Parse the Request
Identify from the user's message:
- **Category**: which post type (`autopsy`, `did-you-know`, `prompt-drop`, `prompt-pattern`, `infographic`, `user-story`)
- **Topic**: the subject to build the post around
- **ICP**: which audience profile to target. Default: `solo-builder`
- **Platform**: Default: `instagram`

### 2. Load Context (read all 3 config files)
Before generating ANY content, read these files in order:
1. `config/brand.yaml` — product info, positioning, voice, differentiators
2. `config/platforms/{platform}.yaml` — platform constraints, category rules, formatting guidelines
3. `config/icps/{icp}.yaml` — target audience profile, pain points, language patterns

### 3. Load the Prompt Template
Read the generation instructions at:
```
prompts/{platform}/{category}.md
```
This file defines the exact output structure, slide-by-slide or section-by-section.

### 4. Generate Content
Using the loaded brand context, platform constraints, ICP profile, and prompt template instructions — generate the full post content. Follow the template structure exactly.

**Interactive categories (e.g., `autopsy`)**: Some categories have a two-step workflow:
- **Step 1**: Generate the initial content (bad prompt, dissection slides) and **pause**. Present the bad prompt to the user and ask them to provide the optimized version (from Prompt Optimizer).
- **Step 2**: Once the user provides the optimized prompt, insert it into the "Optimized Version" slide exactly as given, then complete the remaining sections (payoff slide, caption, hashtags, alt text).

Check the prompt template for whether it specifies an interactive workflow.

### 5. Write Output
Save the generated content as a markdown file to:
```
output/{platform}/{category}/{YYYY-MM-DD}_{slug}.md
```
Where:
- `{YYYY-MM-DD}` is today's date
- `{slug}` is the topic converted to lowercase kebab-case (e.g., "fix my code" → "fix-my-code")

If a file with the same name already exists, append a number: `fix-my-code-2.md`

### 6. Generate Slide Images

After writing the markdown file, generate slide PNGs using the **template-driven renderer**:

```
python3 scripts/render_carousel.py templates/{category}.yaml content.yaml [slug]
```

This uses HTML/CSS templates (rendered via Playwright) for high-quality typography. Output goes to `generated_slides/{slug}/` with one PNG per slide.

**Content can be provided two ways:**
- **YAML content file**: A YAML with placeholder values per slide (see `templates/autopsy/sample_content.yaml` for the format)
- **Markdown post file**: The standard output markdown — the renderer auto-extracts slide content

**Legacy renderer** (Pillow-based, still available):
```
python3 scripts/generate_slides.py output/{platform}/{category}/{filename}.md
```

## Output Markdown Format

Every generated post MUST use this frontmatter + section structure:

```markdown
---
platform: instagram
category: autopsy
topic: "fix my code"
icp: solo-builder
date: 2026-03-14
status: draft
---

# [Post Title]

## Slides
(for carousels — one subsection per slide)

### Slide 1 — [Slide Title]
[slide content]

### Slide 2 — [Slide Title]
[slide content]

...

## Image Content
(for single images — the text that appears on the image)

## Caption
[Instagram caption text]

## Hashtags
[space-separated hashtags]

## Alt Text
[Accessibility descriptions]
```

## Rules

1. **Always read configs before generating.** Never generate from memory alone — the config files are the source of truth.
2. **Follow the prompt template structure exactly.** The templates define slide counts, section order, and content rules.
3. **Voice consistency.** Match the tone defined in `config/brand.yaml`. Confident, technical but accessible, slightly irreverent. Never preachy or hypey.
4. **ICP targeting.** Content must speak to the specific ICP's pain points and language patterns. Don't write generic content.
5. **Product mentions are minimal.** Most content is educational. Product branding is subtle — typically only on the final slide or as a small CTA.
6. **For autopsy posts: the optimized prompt comes from the user.** Never generate the optimized prompt yourself — always pause and ask the user to provide it from Prompt Optimizer. Use it exactly as given.
7. **For infographics: use NotebookLM as the research backbone.** Query NotebookLM for factual, research-backed content before generating. See `prompts/instagram/infographic.md` for the integration workflow.
8. **For user stories: never fabricate testimonials.** All stories must come from real user experiences provided by the user.

## Carousel Template System

Templates are defined in `templates/`. Each template has:
- **`templates/{name}.yaml`** — Declares slide types, order, and named placeholders
- **`templates/{name}/`** — HTML/CSS files per slide type (one `.html` per slide type + `base.css`)

### Available Templates

| Template | Dir | HTML Built | Status |
|----------|-----|------------|--------|
| Prompt Autopsy | `templates/autopsy/` | Yes | Ready |
| Did You Know | `templates/did-you-know/` | No | YAML only |
| Daily Prompt Drop | `templates/prompt-drop/` | No | YAML only |
| Prompt Pattern | `templates/prompt-pattern/` | No | YAML only |
| Prompt Infographic | `templates/infographic/` | No | YAML only |
| User Story | `templates/user-story/` | No | YAML only |

### Template YAML Structure
The YAML defines each slide's `type` (maps to an HTML file), its `placeholders` (what the AI fills in), and whether it `repeat`s (like dissection slides).

### Content YAML Format
To fill a template, provide a YAML with values for each placeholder:
```yaml
hook:
  prompt_text: "the bad prompt"
  hook_line: "the hook line"
dissection_1:
  title: "Failure Name"
  body: "Why this fails"
```

### Creating a New Carousel Template
1. Create `templates/{name}.yaml` with slide definitions and placeholders
2. Create `templates/{name}/base.css` with shared styles
3. Create one `templates/{name}/{slide_type}.html` per slide type
4. Use `{{placeholder_name}}` markers in the HTML where content goes
5. Test: `python3 scripts/render_carousel.py templates/{name}.yaml content.yaml`

## NotebookLM Integration (Infographics)

Infographic content is research-backed via NotebookLM:
1. Maintain a NotebookLM notebook with prompt engineering sources (model docs, PO guides, research)
2. Query NotebookLM for factual content on the infographic topic
3. Format the output into the infographic template with brand voice
4. See `prompts/instagram/infographic.md` for the full workflow

## Buffer Integration (Phase 2)

Currently, generated posts are saved as markdown files for manual review and upload to Buffer.

When Buffer API token is configured in `config/buffer.yaml`:
- **Template fetching**: Pull Buffer templates to discover new post formats
- **Scheduling**: Push generated content directly to Buffer as draft posts
